# Concept: Filter AST to HTML Debugger

## 1. Goal

The primary goal is to create a new function that leverages the Abstract Syntax Tree (AST) generated by `parseFilter()` to produce a structured and interactive HTML representation of the original filter string. This HTML output will serve as a debugging and educational tool, allowing users to visually inspect the components of a filter and understand how TiddlyWiki parses it.

Each component of the filter (runs, prefixes, operators, operands, etc.) will be wrapped in a `<span>` element with specific CSS classes and a `title` attribute. The `title` attribute will provide a human-readable description of the component on hover.

## 2. Input and Output

*   **Input:** A standard TiddlyWiki filter string (e.g., `[tag[HelloThere]] -[search:title,caption{my-tiddler}]`).
*   **Output:** An HTML string that visually represents the parsed filter.

## 3. Proposed Function

A new wikimethod could be created, for example `parseFilterToHtml(filterString)`.

```javascript
/*
Parses a filter string and returns an HTML representation.
*/
exports.parseFilterToHtml = function(filterString) {
    var ast = this.parseFilter(filterString);
    var html = "";
    // ... logic to traverse AST and build HTML string ...
    return html;
};
```

## 4. AST-to-HTML Mapping and Traversal Logic

The function will traverse the AST array produced by `parseFilter()`.

### 4.1. `operation` Array (Top Level)

The function will iterate through each `operation` object in the main array. Each `operation` will be wrapped in a container element.

*   **HTML:** `<div class="tc-filter">...</div>`
*   **CSS Class:** `tc-filter`

### 4.2. `operation` Object

For each `operation`, a wrapper `<span>` will be created.

*   **HTML:** `<span class="tc-filter-run">...</span>`
*   **CSS Class:** `tc-filter-run`
*   **Tooltip:** "A filter run, which is a sequence of operators applied to a set of titles."

### 4.3. `operation.prefix`

If a prefix exists (`+`, `-`, `~`, `=`, `=>`, or a named prefix), it will be rendered inside its own `<span>`.

*   **HTML:** `<span class="tc-filter-prefix tc-filter-prefix-and" title="Prefix: 'and'">+</span>`
*   **CSS Classes:** `tc-filter-prefix` and a modifier like `tc-filter-prefix-and`, `tc-filter-prefix-except`, `tc-filter-prefix-else`, etc.
*   **Tooltip:** "Prefix: 'and' - Intersects the results of this run with the main results."

### 4.4. `operation.operators` Array

The function will then iterate through the `operators` array within each `operation`.

*   A single operator will be wrapped. For `[tag[foo]]`:
    *   **HTML:** `<span class="tc-filter-operator" title="Operator: tag">...</span>`
    *   **CSS Class:** `tc-filter-operator`
    *   **Tooltip:** "Operator: 'tag' - Filters for tiddlers with the specified tag."
*   The literal brackets `[` and `]` will be included to maintain the visual structure.

### 4.5. Operator Name and Suffixes

The rendering will distinguish between the operator name and its structured suffixes. The parser provides a detailed `operator.suffixes` array, which is an array of "suffix groups". Each group is an array of "suffix entries". This structure will be mapped to a nested HTML structure for detailed inspection.

*   **AST `operator.suffixes` for `:op-purpose,op-output:literal`:** `[['op-purpose', 'op-output'], ['literal']]`

*   **Operator Name HTML:**
    *   `<span class="tc-filter-operator-name" title="Operator Name: search">search</span>`

*   **Suffixes HTML:** The logic will iterate through the `suffixes` array. Each group will be a `<span>`, and each entry within it will be another `<span>`. The separating colons and commas will be preserved.

    ```html
    <span class="tc-filter-suffix-group" title="Suffix Group">
        :<span class="tc-filter-suffix-entry" title="Suffix Entry: op-purpose">op-purpose</span>,<span class="tc-filter-suffix-entry" title="Suffix Entry: op-output">op-output</span>
    </span>
    <span class="tc-filter-suffix-group" title="Suffix Group">
        :<span class="tc-filter-suffix-entry" title="Suffix Entry: literal">literal</span>
    </span>
    ```

### 4.6. `operator.operands` Array

Finally, the operands are rendered. The markup will differ based on the operand type.

*   **Normal Operand** (`[tag[HelloThere]]`):
    *   **HTML:** `<span class="tc-filter-operand" title="Operand (Literal)">HelloThere</span>`
    *   **CSS Class:** `tc-filter-operand`
*   **Indirect Operand** (`{my-tiddler}`):
    *   **HTML:** `<span class="tc-filter-operand tc-filter-operand-indirect" title="Operand (Indirect from Tiddler: my-tiddler)">...</span>`
    *   **CSS Classes:** `tc-filter-operand tc-filter-operand-indirect`
*   **Variable Operand** (`<my-variable>`):
    *   **HTML:** `<span class="tc-filter-operand tc-filter-operand-variable" title="Operand (From Variable: my-variable)">...</span>`
    *   **CSS Classes:** `tc-filter-operand tc-filter-operand-variable`

## 5. Example Walkthrough

**Filter String:**
`[is[current]] +[tag{!!title}]`

**Simplified AST:**
```json
[
    {
        "prefix": "",
        "operators": [
            {"operator": "is", "operands": [{"text": "current"}]}
        ]
    },
    {
        "prefix": "+",
        "operators": [
            {"operator": "tag", "operands": [{"text": "!!title", "indirect": true}]}
        ]
    }
]
```

**Generated HTML (conceptual):**

```html
<div class="tc-filter">
  <span class="tc-filter-run" title="A filter run...">
    <span class="tc-filter-operator" title="Operator: is">
      [<span class="tc-filter-operator-name" title="Operator Name: is">is</span>[<span class="tc-filter-operand" title="Operand (Literal)">current</span>]]
    </span>
  </span>
  <span class="tc-filter-run" title="A filter run...">
    <span class="tc-filter-prefix tc-filter-prefix-and" title="Prefix: 'and'">+</span>
    <span class="tc-filter-operator" title="Operator: tag">
      [<span class="tc-filter-operator-name" title="Operator Name: tag">tag</span>{<span class="tc-filter-operand tc-filter-operand-indirect" title="Operand (Indirect from Tiddler: !!title)">!!title</span>}]
    </span>
  </span>
</div>
```

## 7. JavaScript Macro Concept

A global macro will make this functionality accessible from wikitext.

*   **Name:** `filter-html`
*   **Parameter:** `filter` (the filter string to parse)
*   **Wikitext Usage:** `<<filter-html "[tag[HelloThere]] +[sort[title]]">>`

### 7.1. Macro Definition (`filter-html.js`)

The macro will be defined in a new file, `core/modules/macros/filter-html.js`.

```javascript
/*\ntitle: $:/core/modules/macros/filter-html.js
type: application/javascript
module-type: macro

Macro to return a syntax-highlighted HTML representation of a filter string.
\*/
"use strict";

exports.name = "filter-html";

exports.params = [
    { name: "filter" }
];

/*
Run the macro
*/
exports.run = function(filter) {
    return this.wiki.parseFilterToHtml(filter || "");
};
```

### 7.2. Tiddler for CSS

A new tiddler will be created at `core/css/filter-html.css.tid` containing the styles.
